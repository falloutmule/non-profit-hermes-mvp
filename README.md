# Non-Profit Hermes MVP

Board-visible website and operations scaffold for a small nonprofit, mutual-aid group, or volunteer-run charity. Captures messy human input from Telegram, structures it into Google Sheets records, schedules dated commitments in Google Calendar, and publishes an approved-safe public snapshot to GitHub Pages.

**Repository:** [falloutmule/non-profit-hermes-mvp](https://github.com/falloutmule/non-profit-hermes-mvp)
**Public site:** https://falloutmule.github.io/non-profit-hermes-mvp/
**Current status:** see [PROJECT_STATUS.md](PROJECT_STATUS.md)

---

## Purpose and privacy boundary

This system helps a small operations team:

- capture needs, donations, reports, tasks, inventory, and event drafts from Telegram;
- store durable records in Google Sheets;
- schedule real commitments in Google Calendar (only when explicitly approved);
- publish a filtered, board-safe snapshot to GitHub Pages.

**Intended privacy boundary:** raw private intake should never be published, and all public-facing content should pass approved-safe filtering and human review. CLEANUP-002 resolved the prior export-safety P0 gaps (schema divergence, missing consent/status gates, unescaped HTML, and board-log ID exposure). Publication remains frozen because `/daily` still couples summary invocation to generation behavior; this remaining P0 is scheduled for CLEANUP-003. See [SECURITY_AND_PRIVACY.md](SECURITY_AND_PRIVACY.md) and [PROJECT_STATUS.md](PROJECT_STATUS.md) for the current gap list.

## Architecture

```
Telegram chat
    │
    ▼
Hermes plugins (external, per-user install)
    │  register /daily /need /donation /report /task /inventory /event
    ▼
scripts/telegram_intake_router.py    ← draft-first intake, follow-up routing,
    │                                  active-draft state, /daily summary
    ▼
scripts/non_profit_hermes_ops.py     ← backend: Google Sheets writes + Calendar
    │                                  create + AuditLog for every mutation
    ▼
Google Sheets (system of record)     ← Requests, Donations, Reports, Tasks,
    │                                  Inventory, CalendarLog, AuditLog
    ▼
scripts/sync_approved_safe_data.py   ← reads Sheets/Calendar, filters to
    │                                  approved-safe, writes docs/ static site
    ▼
docs/ (GitHub Pages, main /docs)     ← static HTML + JSON, public
```

See [ARCHITECTURE.md](ARCHITECTURE.md) for the full component map, data flow, and privacy boundaries.

## Durable data store

**Google Sheets is the system of record.** Every intake becomes a structured row. Tabs:

| Tab | Purpose |
|-----|---------|
| Requests | Community needs |
| Donations | Incoming donations |
| Reports | Activity/impact reports |
| Tasks | Follow-up tasks (internal only) |
| Inventory | Stock levels (internal only) |
| CalendarLog | Calendar event drafts + promotion records |
| AuditLog | Every create/update/delete, append-only |

**Google Calendar** is used only for dated commitments. Calendar creation from `/event` is **disabled** until EVENT-004.

## Generated public site (`docs/`)

The active public site is static HTML/JSON under `docs/`. It is generated by the sync script, not hand-edited.

```
docs/
  .nojekyll
  index.html          ← home page
  today.html          ← daily brief snapshot
  current-needs.html  ← approved needs
  calendar.html       ← approved public events
  reports.html        ← approved reports
  deployment-proof.html
  data/               ← approved-safe JSON exports
```

Pages source is `main /docs` (verified via GitHub API). Root-level `.md` files and `_config.yml` are **legacy Jekyll artifacts** and do not represent the current site.

## Local plugins vs repository code

The actual Telegram plugins live **outside this repository**, under the local Hermes plugin directory:

```
C:\Users\<user>\AppData\Local\hermes\plugins\
  non-profit-hermes-daily/
  non-profit-hermes-need/
  non-profit-hermes-donation/
  non-profit-hermes-report/
  non-profit-hermes-task/
  non-profit-hermes-inventory/
  non-profit-hermes-event/
```

Each plugin is a thin shim: it registers the command and delegates to `scripts/telegram_intake_router.py`. The repository contains the router, backend, sync script, tests, and documentation — not installable plugin copies. Plugin reproducibility from GitHub is a planned cleanup item (CLEANUP-005).

## Setup prerequisites

- Python >= 3.11
- Google Cloud project with Sheets and Calendar APIs enabled
- OAuth credentials (`google_token.json`) with Sheets and Calendar scopes
- Hermes Agent running with Telegram gateway

**Warning:** This project does not yet have a `requirements.txt` or `pyproject.toml`. Current dependencies:

- `google-auth`
- `google-api-python-client`
- `pytest` (test only)

Dependency pinning is a planned cleanup item (CLEANUP-005).

## Test commands

```
python -m pytest -q
python -m py_compile scripts/*.py
git diff --check
```

Current CLEANUP-002 suite: 53 passed, 52 subtests passed.

### ⚠ Live-write test mode

`python -m pytest` is an offline, fake-based test suite and makes no live Google calls. **`scripts/telegram_intake_router.py --test` initializes the live Google Sheets and Calendar services**; do not run it unless live-service access is intended. **`scripts/non_profit_hermes_ops.py --test-write` is an explicit live-write operational mode.** Do not run it unless you intend to create test records in the live Sheet.
